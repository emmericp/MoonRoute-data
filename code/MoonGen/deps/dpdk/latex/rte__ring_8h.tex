\subsection{Detailed Description}
R\+T\+E Ring

The Ring Manager is a fixed-\/size queue, implemented as a table of pointers. Head and tail pointers are modified atomically, allowing concurrent access to it. It has the following features\+:


\begin{DoxyItemize}
\item F\+I\+F\+O (First In First Out)
\item Maximum size is fixed; the pointers are stored in a table.
\item Lockless implementation.
\item Multi-\/ or single-\/consumer dequeue.
\item Multi-\/ or single-\/producer enqueue.
\item Bulk dequeue.
\item Bulk enqueue.
\end{DoxyItemize}

Note\+: the ring implementation is not preemptable. A lcore must not be interrupted by another task that uses the same ring.

Create a new ring named {\itshape name} in memory.

This function uses {\ttfamily memzone\+\_\+reserve()} to allocate memory. Then it calls \hyperlink{rte__ring_8h_a7a44f25853c926ac1b29562857244366}{rte\+\_\+ring\+\_\+init()} to initialize an empty ring.

The new ring size is set to {\itshape count}, which must be a power of two. Water marking is disabled by default. The real usable ring size is {\itshape count-\/1} instead of {\itshape count} to differentiate a free ring from an empty ring.

The ring is added in R\+T\+E\+\_\+\+T\+A\+I\+L\+Q\+\_\+\+R\+I\+N\+G list.
